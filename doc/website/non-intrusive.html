The cost of logging

Often, logging is used for a debugging purpose.

  • impact on execution time. This is the more obvious consequence.
  • impact on executable size
  • impact on thread ordonancement: can easily generate a switching context and alter the classical execution flow. A multithreaded library can slow down all the threads due to logging synchronisation, and augment the time spent in a critical section if information are logged there, thus slowing down the whole application. Can participate in the apparition of heisenbugs.
  • impact on memory usage. Can interfer with memory leak detection libraries and participate in the apparition of heisenbugs.
  • impact on source code confidenciality: can give losts of information (that's the goal!), like source file tree, function names, ... Adds a lot of literal strings
  • impact on stability: if throws exceptions, contains bugs, ...
  • impact on safety: if use unsafe functions, does improper input validation, contains bugs like buffer overrun, ...
  • impact on portability: if use copiler/OS specific code
  • impact on maintenability: if adds dependencies
