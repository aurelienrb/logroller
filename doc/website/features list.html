Logging features list

source code features
  • need initialisation call
  • scope logging
  • UNICODE support
  • Blob support
  • exception safe library
  • make sur the logging requests never throw an exception
  • if a logging requests throws an exception, the logging request already competed is still saved
  • automatically collects __FILE__, __LINE__ and __FUNCTION__
documentation filtering
  • filtering can be done at compile time with macros
  • filtering can be done at compile time without macros
  • filtering can be done at runtime
  • filtering can be done via message levels
  • filtering can be done via message type
  • filtering can be done via message location (source file name...)
  • filtering can be done via regular expressions
  • filtering can be customized
log files
  • automatically generate a log file name
  • Ability to output the log messages in different files depending on their type (error.log, warning.log, ...)
  • generated log file name does not conflict if several instances of the same program are running
log output and formatting
  • can send the log messages to different output at the same type (multiplexed output)
  • possibility to give its own output type
  • possibility to format data in a custom way
output types already provided
  • file
  • socket
  • pipe
  • database
  • debugger
  • shared memory
formatting format already provided
  • plain text
  • html
  • xml
  • syslog
  • binary
macros
  • library has been designed to be usable without macros
multithreading
  • pthread
runtime configuration supported compilers
supported platforms
  • optimized for 32bit architecture
  • optimized for 64bit architecture
license usable in C COMPILATION AND INSTALLATION
  • minimal source code size
  • no external dependencies (full C++ standard code)
  • makefile included
extra features
  • log application crash
  • memory debuging (tracing malloc/new...)
  • designed to work in restricted environement (embedded, real time, application crash)
  • integrate with error management framework
  • integrate with unit testing framework
* Transparent Logging to file / console / network. * Supports different logging levels. * Customizable : Log level, Module name, etc., can be configured on per file basis. * Zero Performance overhead when the logging is disabled. * Simple APIs similar to printf()
