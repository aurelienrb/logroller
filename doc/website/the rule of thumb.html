My rules of thumb for a good logging library

  • Simplicity
  • Readability (cool syntax)
  • No source code overbloating
  • easy customisation
  • easy integration, compilation
  • very open license
  • do not expose implementation details
  • well documented
  • almost no learning curve
  • safely usable by a complete C++ beginner
  • simple and stupid by design (no smart technique to solve bad design issues)
  • Flexibility (open architecture)
  • help in writing better, more robust code, not the opposite
  • do not impose the use of unwanted features
  • do not make the user pay for unused features
  • be able to run in a low resource environement
